Invigilator

1. Introduction

1.1 Purpose

The purpose of this document is to present a detailed description of the open-source software Invigilator. It will explain the purpose and features of the software, the interfaces of the software, what the software will do and the constraints under which it must operate. This document is intended for users of the software and also potential developers.

1.2 Document Conventions

This Document was created based on the IEEE template for System Requirement Specification Documents.

1.3 Intended Audience and Reading Suggestions

Educators to take online tests and students who will be attending the online tests.

1.4 Production Scope

Invigilator is web application for taking online tests. It can be used for supervising and marking the exams more efficiently. There will be ways of answering questions in many ways. It will also benefit the review process of the tests.

2. Overall Description

2.1 Product Perspective

The Invigilator web application aims to simplify online tests, benefiting both examiners and examinees, while also ensuring the integrity of the examination process. It saves time, enhances efficiency, and provides a user-friendly experience, contributing to the effectiveness of online education.

2.2 Product Functions

  • There should be a profile for examiners
  • There should be a profile for examinees
  • Multiple types of answering systems
  • Automatic evaluation for closed type answers
  • Exams availability can be limited by the examiner
  • Exam can have a set duration
  • There will be review system for an evaluated exam
  • Their will certain restrictions to examinees device for the duration of the exam
  • The webcam should be used to keep an eye on the examinee
  • Seamless switch between devices
